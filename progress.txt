### Architecture decisions
- **Two declarations per enum**: Every enum produces both a type alias and a Known enum,
  following the legacy emitter's pattern for output parity. The type alias is the primary
  type (referenced via `typeRefkey(enum)`), while the Known enum documents known values
  (referenced via `knownValuesRefkey(enum)`).
- **Refkey usage**: `typeRefkey(enum)` is on the type alias, `knownValuesRefkey(enum)` on

  linking to the type alias via `{@link}`. Members get `member.doc` or fall back to the
  literal value string.

- Native TypeSpec enums are always `isFixed=true` in TCGC. Extensible enums require
  Azure-specific union-as-enum patterns. Tests simulate extensibility by overriding isFixed.
- Alloy's `<EnumMember jsValue={...}>` correctly renders string values in quotes and
  numeric values bare, matching TypeScript enum syntax.

  implementation plan's convention.

- Multi-level hierarchies may include transitive subtypes in the map — always filter
  by `baseModel === model` for direct subtypes only.
- Test assertions must match actual TCGC output, not assumed TypeSpec properties.
