### Pattern for writing tests
1. Import `"@alloy-js/core/testing"` (side-effect import registers matchers)
2. Use `Tester.createInstance()` then `runner.compile(t.code\`...\`)` to compile TypeSpec
3. Wrap components in `<TestFile program={program}>` for rendering
4. Assert with `expect(template).toRenderTo(\`expected output\`)`

  - Providing sdkPackage data to child components

  - Throwing when used without provider
  - Consuming via raw `useContext()`
  - Exposing full sdkPackage object
- All 11 tests pass (5 infrastructure + 6 context).

### Architecture decisions
- Context value exposes `sdkPackage`, `clients`, `models`, `enums`, `unions` as typed accessors.
- `program` is NOT duplicated in our context — it's already available via `useTsp()` from
  `@typespec/emitter-framework`.
- TCGC `SdkContext` type is aliased as `TcgcSdkContext` to avoid name collision with our
  Alloy named context.

### Key learnings (see knowledge.md)
- TCGC prunes unreferenced models — always include operations that reference types in tests.
- Alloy JSX is lazy — use `renderToString()` to trigger throws in error-path tests.
- `EmitContext` requires a `perf` property — provide no-op mock in tests.
